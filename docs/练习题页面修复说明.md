# 📝 练习题页面修复说明

## 🐛 问题描述

**原问题**：点击"练习题"标签无反应，F12显示错误：
```
TypeError: Cannot set properties of null (setting 'innerHTML')
```

**原因**：在 `poem_study.html` 中尝试在当前页面显示练习题，但缺少对应的DOM元素 `exercisesContent`。

## ✅ 解决方案

### 1. 创建独立的练习题页面

新建 `templates/exercises.html` - 专门的练习题页面，包含：

**功能特性**：
- ✅ 美观的卡片式布局
- ✅ 题号和题型标签
- ✅ 4个选项的单选题
- ✅ 点击选择，高亮显示
- ✅ 提交答案并自动判分
- ✅ 显示正确答案和详细解析
- ✅ 成绩统计（正确数、正确率、得分）
- ✅ 重新作答功能
- ✅ AI生成新题功能

**页面路由**：
```
/exercises?poem_id={poem_id}
```

### 2. 修改跳转逻辑

修改 `poem_study.html` 中的 `showExercises()` 函数：

**修改前**：
```javascript
// 在当前页面显示练习题
async function showExercises() {
    const exercisesDiv = document.getElementById('exercisesContent');
    // ... 复杂的逻辑
}
```

**修改后**：
```javascript
// 跳转到练习题页面
function showExercises() {
    if (!authToken) {
        alert('请先登录后再做练习');
        return;
    }
    window.location.href = `/exercises?poem_id=${poemId}`;
}
```

### 3. 添加路由

在 `app.py` 中添加练习题页面路由：

```python
@app.route('/exercises')
def exercises_page():
    from flask import render_template
    return render_template('exercises.html')
```

## 🔧 技术细节

### 页面通信

通过URL参数传递古诗ID：
```javascript
// 跳转时携带参数
window.location.href = `/exercises?poem_id=${poemId}`;

// 目标页面接收参数
const urlParams = new URLSearchParams(window.location.search);
const poemId = urlParams.get('poem_id');
```

### 数据流程

1. **加载练习题**
   ```
   GET /api/exercises/poem/{poem_id}
   → 返回题目列表（不含答案）
   ```

2. **提交答案**
   ```
   POST /api/exercises/submit
   {
     exercise_id: 1,
     user_answer: "A",
     time_spent: 30
   }
   → 返回判分结果和解析
   ```

3. **生成新题**
   ```
   POST /api/exercises/generate/{poem_id}
   {
     question_type: "主旨理解"
   }
   → AI生成新题目
   ```
