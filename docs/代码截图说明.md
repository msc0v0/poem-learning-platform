@auth_bp.route('/login', methods=['POST'])
def login():
    try:
        data = request.get_json()
        
        if not data or not data.get('username') or not data.get('password'):
            return jsonify({'error': 'Username and password are required'}), 400
        
        # 查找用户
        user = User.query.filter_by(username=data['username']).first()
        
        if not user or not user.check_password(data['password']):
            return jsonify({'error': 'Invalid username or password'}), 401
        
        if not user.is_active:
            return jsonify({'error': 'Account is deactivated'}), 401
        
        # 更新最后登录时间
        user.last_login = datetime.utcnow()
        db.session.commit()
        
        # 生成token
        token = user.generate_token()
        
        return jsonify({
            'message': 'Login successful',
            'token': token,
            'user': user.to_dict()
        }), 200
        
    except Exception as e:
        return jsonify({'error': str(e)}), 500

### 4. 代码截图：前端认证请求处理

**文件**：`templates/poems_list.html`

**截取代码段**（第 629-657 行）：
```javascript
async function handleLogin(e) {
    e.preventDefault();
    
    const username = document.getElementById('loginUsername').value;
    const password = document.getElementById('loginPassword').value;
    
    try {
        const response = await fetch(`${API_BASE}/auth/login`, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ username, password })
        });
        
        const data = await response.json();
        
        if (response.ok) {
            authToken = data.token;
            localStorage.setItem('authToken', authToken);
            currentUser = data.user;
            updateUserUI();
            hideLogin();
            alert('登录成功！');
        } else {
            alert('登录失败: ' + data.error);
        }
    } catch (error) {
        alert('登录错误: ' + error.message);
    }
}
```

**图注**：图 2-4 展示了前端登录请求处理逻辑，JWT 令牌存储在 localStorage 中用于后续 API 调用。

## 截图技巧

### 代码截图建议：
1. 使用 VS Code 或其他 IDE 的截图功能
2. 确保代码高亮显示清晰
3. 包含行号以便引用
4. 调整字体大小确保可读性（建议 12-14pt）
5. 截图格式：PNG（高清）

### 网页截图建议：
1. 使用浏览器的截图工具或 Snipping Tool
2. 确保 UI 元素完整显示
3. 分辨率至少 1920x1080
4. 可以使用浏览器开发者工具调整窗口大小以获得最佳展示效果
